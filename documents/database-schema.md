# Database Schema Documentation

This document summarizes the Entity-Relationship structure implemented in the Prisma schema for the AI Recruitment System.

## Domain 1: Companies & Departments
- **Company**: Stores company profiles, branding, and details.
- **Department**: Divisions within a company (e.g., HR, IT).

## Domain 2: Users & Roles
- **User**: Core authentication and general user info.
- **Role**: Defines roles (`ADMIN`, `RECRUITER`, `CANDIDATE`).
- **UserRole**: Associates users with roles.
- **RecruiterProfile**: Extra fields for users with `RECRUITER` role.
- **Candidate**: Extra fields for users with `CANDIDATE` role.

## Domain 3: Jobs & Skills
- **JobPosting**: Stores job advertisements, requirements, salary ranges.
- **Skill**: Core skills dictionary (e.g., React, Java, Communication).
- **SkillAlias**: Alternative names mapping to core skills (e.g., ReactJS -> React).
- **JobSkill**: Many-to-many relationship tracking which skills are required for which job.

## Domain 4: Candidates & Resumes
- **Resume**: Records uploaded CV files and their storage paths in Supabase Storage.
- **ResumeParsedData**: Holds JSON extraction data parsed from the CV by the AI Service.
- **CandidateSkill**: Many-to-many relationship extracting candidate skills from their resume.

## Domain 5: Applications & Processing
- **CvUploadBatch**: Tracks batch imports of multiple CVs.
- **Application**: Tracks a candidate's journey applying for a specific job.
- **ApplicationStatusHistory**: Audit log of stage transitions for an application.
- **AiMatchingResult**: Output from the AI Service matching a Resume to a JobPosting, including scores and explanations.

## Domain 6: Interviews & Communication
- **Interview**: Scheduled interviews, types, status, and scores.
- **InterviewQuestion**: Suggested questions for interviews generated by AI.
- **Notification**: Alerts delivered to users regarding application statuses or system events.

## Domain 7: System & Telemetry
- **UnrecognizedSkill**: Logs skills extracted from CVs that do not exist in the dictionary, awaiting admin review.
- **AuditLog**: Generic logging of critical actions for system compliance.
- **InfrastructureHealthRecord**: Used for DB health checks.

> For exact types, relationships, and PostgreSQL naming mappings, please refer directly to the `backend/prisma/schema.prisma` file.
